# /sdd-poc Command

불확실한 기술적 가설을 체계적으로 실험하고 판정하는 SDD PoC 명령입니다.

## 사용법

```
/sdd-poc [SP-NNN 또는 주제]        # PoC 실험 시작
/sdd-poc [SP-NNN] result           # 실험 결과 정리 + 판정
```

## 실행 흐름

### Phase 1: 문제 정의

1. **핵심 문제** 1줄로 정의 (흔들리면 안 됨)
2. **성공 기준** 측정 가능하게 정의 (예: "4장 중 3장 이상 캐릭터 구분 성공")
3. **제약 조건** 명시 (GPU VRAM, 모델, 환경)

```
## 문제: [1줄]
## 성공 기준: [측정 가능]
## 제약: [환경]
```

### Phase 2: 리서치

1. 기존 코드/메모리에서 이미 시도한 것 확인
2. Gemini 브레인스톰으로 후보 전략 수집
3. 후보를 **실현 가능성 × 기대 효과** 매트릭스로 정리
4. **최대 5개** 가설로 제한

```
| # | 가설 | 실현성 | 기대효과 | 테스트 순서 |
|---|------|:------:|:-------:|:---------:|
| A | ...  | 상     | 상      | 1         |
| B | ...  | 중     | 상      | 2         |
```

### Phase 3: 베이스라인

1. **현재 방식(기존)으로 먼저 기준점 생성** — 이거 없으면 비교 불가
2. 베이스라인 결과를 저장 + 기록
3. 성공 기준으로 베이스라인 점수 매기기

### Phase 4: 순차 실험

**규칙:**
- 한 번에 **변수 1개만** 변경
- 베이스라인과 **동일한 시나리오/시드**로 테스트
- 각 실험 결과를 즉시 기록 (파일명 컨벤션: `poc_{주제}_{전략}_{seed}`)
- 실험 당 최소 **4샘플** (시드 4개)

```
실험 A: [전략명]
  변경점: [베이스라인에서 뭘 바꿨는지]
  결과: [4장 중 N장 성공]
  판정: 성공/실패/부분성공
```

### Phase 5: 조합 실험 (선택)

- 개별 실험에서 성공한 전략끼리 조합
- 조합 시에도 **한 번에 1개 추가**
- 조합이 오히려 나빠지면 즉시 폐기

### Phase 6: 판정

| 판정 | 기준 |
|------|------|
| **채택** | 성공 기준 충족 + 베이스라인 대비 명확한 개선 |
| **조건부 채택** | 부분 성공, 추가 튜닝으로 개선 가능 |
| **보류** | 가능성은 있으나 현재 환경에서 제약 |
| **폐기** | 베이스라인보다 나쁘거나 실현 불가 |

### Phase 7: 결과 문서화

실험 결과를 `.claude/tasks/current/SP-NNN_*/poc.md`에 저장:

```markdown
# SP-NNN PoC 결과

## 문제: [1줄]
## 성공 기준: [측정 가능]

## 실험 결과

| 전략 | 샘플 수 | 성공률 | 판정 | 비고 |
|------|:------:|:-----:|:----:|------|
| 베이스라인 | 4 | 50% | 기준 | |
| A | 4 | 75% | 채택 | |
| B | 4 | 25% | 폐기 | |

## 최종 결론
[채택된 전략 + 다음 단계]
```

## 산출물 관리

태스크 디렉토리에 결과를 함께 관리:

```
.claude/tasks/current/SP-NNN_*/
├── spec.md
├── poc.md                    ← git push (실험 결과 문서)
├── poc_artifacts/
│   └── workflows/            ← git push (JSON, 가벼움)
│       ├── baseline.json
│       └── strategy_a.json
└── .gitignore                ← samples/ 제외
```

- `poc.md` + `workflows/` → **git push** (텍스트/JSON)
- **이미지 샘플** → MinIO 스토리지에 업로드, poc.md에 URL 링크
  - 경로: `poc/{SP-NNN}/{전략}_{seed}.png`
  - poc.md에서 참조: `[baseline s100](http://localhost:9000/shorts-producer/poc/SP-NNN/baseline_s100.png)`

## 리서치 태스크 완결

PoC 결과 자체가 DoD인 태스크 (코드 구현 없이 실험 결론이 산출물):

| 판정 | 태스크 처리 |
|------|-----------|
| **채택** | spec을 구현 태스크로 전환 (design → sdd-run) |
| **조건부 채택** | poc.md에 조건 명시 → design에 반영 |
| **보류** | status: `blocked` + 사유 기록 |
| **폐기** | done/으로 이동, poc.md가 최종 산출물 |

## 주의사항

- **문제 정의가 바뀌면 실험 전체를 리셋** — 중간에 목표를 바꾸면 결과가 무의미
- **"기존이 나음"도 유효한 결론** — 억지로 새 방법을 채택하지 않는다
- **감상이 아닌 데이터로 판정** — "좋아 보인다" 금지, 성공 기준 기반 판정
- **실험 중 발견한 부수 인사이트**도 기록 (예: "BREAK가 ComfyUI에서 작동함" 확인)
- 실험 이미지는 `ComfyUI/output/poc_{주제}_*` 네이밍으로 로컬 보존 + MinIO 업로드
