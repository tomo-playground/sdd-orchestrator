# 로드맵: 정밀 연출을 위한 레이어드 태그 시스템 혁신

## 1. 개요 (Vision)
프롬프트를 단순 텍스트가 아닌 **의미론적 데이터 레이어**로 관리하여, 캐릭터의 일관성을 유지하면서도 장면의 세부적인 연출(표정, 구도, 소품 등)을 정밀하게 제어할 수 있는 기반을 구축함.

## 2. 4대 데이터 레이어 정의 (The 4 Layers)

| 레이어 | 정의 | 주요 태그 예시 | 지속성 |
| :--- | :--- | :--- | :--- |
| **Layer 1: Physical** | 캐릭터의 고유 신체 특징 | 성별, 머리색, 눈색, 인종, 체형 | 영구적 |
| **Layer 2: Vibe** | 캐릭터의 기본 기색/인상 | 날카로운 눈매, 처진 눈, 냉정한 분위기 | 준영구적 |
| **Layer 3: Outfit** | 탈부착 가능한 의상 및 소품 | 교복, 수영복, 안경, 모자, 가방, 무기 | 가변적 |
| **Layer 4: Direction** | 특정 장면의 일시적 연출 | 달리기, 윙크함, 클로즈업, 역광 | 일시적 |

---

## 3. 단계별 실행 계획

### Phase 1: 인프라 및 스키마 확장 (Foundation)
- **목표:** 데이터를 정교하게 담을 수 있는 '그릇'을 만듦.
- **작업 내용:**
    - `characters` 테이블에 `physical_tags`, `vibe_tags`, `outfit_tags` 컬럼 추가 (Integer Array).
    - 백엔드 Pydantic 모델(CharacterCreate, Response) 업데이트.
    - 프론트엔드 TypeScript 인터페이스 업데이트.
- **안전장치:** 기존 `identity_tags`, `clothing_tags`는 마이그레이션 완료 전까지 유지.

### Phase 2: 태그 사전 정밀 분류 (Tag Intelligence)
- **목표:** 시스템이 가진 600여 개의 태그에 '의미'를 부여함.
- **작업 내용:**
    - `tags` 테이블에 `layer_type` 컬럼 추가.
    - 기존 태그들을 `group_name` 기반으로 4대 레이어에 자동 배정하는 스크립트 실행.
    - 분류가 모호한 태그들에 대한 관리자 검수 UI 제공.

### Phase 3: 데이터 마이그레이션 (Migration)
- **목표:** 기존 캐릭터들을 새로운 레이어드 구조로 이사시킴.
- **작업 내용:**
    - `identity_tags` -> `physical_tags` + `vibe_tags`로 분리.
    - `clothing_tags` -> `outfit_tags`로 이동.
    - 이사 과정에서 누락된 핵심 태그(예: 성별) 자동 감지 및 경고.
- **검증:** 마이그레이션 전/후 프롬프트 생성 결과 비교 테스트.

### Phase 4: 지능형 프롬프트 조립기 개발 (Smart Composer)
- **목표:** 쪼개진 데이터로 최적의 프롬프트를 만듦.
- **작업 내용:**
    - **레이어별 배치:** `(Quality) + (Physical) + (Vibe) + (Outfit) + (Direction)` 순서로 자동 조립.
    - **충돌 제어:** Vibe 레이어의 '무표정'과 Direction 레이어의 '웃음'이 충돌할 경우의 가중치 처리 로직.
    - **UI 개선:** 캐릭터 편집 모달에서 레이어별로 태그를 관리하는 직관적인 UX 제공.

---

## 4. 기대 효과 (Expected Outcomes)
1. **일관성 확보:** 어떤 연출에서도 캐릭터의 외형(Physical)이 변하지 않음.
2. **정밀 연출:** "안경만 벗기기", "표정만 바꾸기" 등 부분적 연출 제어가 쉬워짐.
3. **자동화 최적화:** 스토리보드 작성 시 AI가 어떤 레이어를 수정해야 하는지 명확히 인지함.
4. **확장성:** 향후 새로운 연출 요소(예: 특수 효과 레이어)를 추가하기 매우 용이함.
