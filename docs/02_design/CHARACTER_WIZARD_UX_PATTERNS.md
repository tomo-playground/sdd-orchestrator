## Overview

캐릭터 생성 인터페이스(Character Creation Interface, CCI)는 게임/AI Art 플랫폼/아바타 빌더 서비스에서 핵심적인 첫 경험입니다. 사용자가 자신을 표현할 캐릭터를 만들어 몰입감과 개인화를 제공하며, 잘못된 UX는 이탈률을 높이고 제품 경험을 저해합니다.

**핵심 원칙**:
- **간결성 우선**: 복잡한 작업을 단순한 단계로 분해
- **진행 상황 가시성**: 사용자가 현재 위치와 남은 단계를 명확히 인지
- **맥락 제공**: 각 단계에서 의사결정에 필요한 충분한 정보 제공
- **실험 허용**: 되돌리기/재시도가 쉬워야 함

## Step-by-Step vs Single Page 패턴

### Step-by-Step (Wizard) 패턴

**언제 사용하나?**
- 옵션이 10개 이상의 카테고리로 나뉠 때
- 단계 간 의존성이 있을 때 (예: 종족 선택 → 체형 옵션 필터링)
- 초보자가 압도되지 않도록 점진적 공개가 필요할 때
- 모바일 화면처럼 공간이 제한적일 때

**장점**:
- 인지 부하 감소: 한 번에 하나의 작업에 집중
- 명확한 진행 흐름: 단계별 완료 → 성취감
- 조건부 로직 구현 용이: 이전 선택에 따라 다음 옵션 필터링

**단점**:
- 정보 비교 어려움: 이전 단계 정보를 확인하려면 뒤로 이동 필요
- 지루함: 10단계 이상 시 압박감/포기율 증가
- 유연성 부족: 특정 단계만 빠르게 수정하기 어려움

**최적 구현**:
```
이상적인 단계 수: 3-5단계
최대 허용 단계: 7-8단계
2단계는 너무 짧고, 10단계 이상은 피로도 증가
```

**예시 흐름**:
1. **기본 설정** (종족/성별/체형)
2. **얼굴** (눈/코/입 형태)
3. **헤어 & 메이크업** (헤어스타일/색상/장식)
4. **의상** (상의/하의/액세서리)
5. **최종 확인** (프리뷰 + 이름 설정)

### Single Page 패턴

**언제 사용하나?**
- 옵션이 5개 이하 카테고리로 간단할 때
- 전문가 사용자가 전체 옵션을 한눈에 비교하고 싶을 때
- 실시간 비교가 중요할 때 (여러 헤어스타일 동시 비교)
- 데스크톱 환경에서 충분한 화면 공간이 있을 때

**장점**:
- 빠른 접근: 모든 옵션이 즉시 보임
- 정보 비교 용이: 스크롤만으로 전체 옵션 탐색
- 유연성: 원하는 섹션만 골라서 수정

**단점**:
- 초보자 압도: 한 화면에 너무 많은 정보
- 모바일 부적합: 스크롤 길이 증가, 터치 오작동 위험
- 복잡한 의존성 구현 어려움

**최적 구현**:
```
카테고리 섹션을 Accordion/Tab으로 그룹화
기본값은 "추천" 섹션만 펼쳐두고 나머지는 접힌 상태
사용자가 원하는 섹션을 선택적으로 확장
```

### 하이브리드 접근 (권장)

**"Progressive Disclosure" 전략**:
- **초기 화면**: 5가지 핵심 카테고리만 Single Page로 표시
- **고급 옵션**: "더 많은 옵션" 버튼 → 펼쳐지는 섹션 또는 별도 단계
- **빠른 시작**: "랜덤 생성" → 결과물 기반으로 세부 조정

## 카테고리 그룹핑 전략

### 논리적 순서

**상위 → 하위 계층**:
1. **기본 골격** (종족/성별/체형) ← 모든 후속 옵션에 영향
2. **얼굴 구조** (이목구비 배치/크기)
3. **색상/텍스처** (피부/눈/헤어 색상)
4. **장식** (헤어스타일/메이크업/문신)
5. **의상** (옷/액세서리)

**근거**: 사용자는 "큰 그림"을 먼저 정하고 세부사항을 조정하려는 경향

### 시각적 중요도 순서

**게임 UI 관례**:
- **Genshin Impact**: 얼굴 → 헤어 → 의상
- **FFXIV**: 종족/부족 → 얼굴형 → 헤어스타일 → 색상 → 세부 조정 (눈/코/입 형태)

**이유**: 캐릭터의 정체성을 가장 많이 드러내는 요소를 우선 배치

### 그룹핑 기법

**시각적 그룹핑**:
```
┌─────────────────────┐
│ BODY (기본)         │
│ • Race              │
│ • Gender            │
│ • Height/Weight     │
└─────────────────────┘
┌─────────────────────┐
│ FACE (얼굴)         │
│ • Face Shape        │
│ • Eyes              │
│ • Nose/Mouth        │
└─────────────────────┘
┌─────────────────────┐
│ HAIR & MAKEUP       │
│ • Hairstyle         │
│ • Hair Color        │
│ • Makeup            │
└─────────────────────┘
```

**색상 코딩**:
- 기본 설정: 파란색 아이콘
- 외형: 초록색 아이콘
- 장식: 보라색 아이콘

### "추천/인기" 표시 방식

**1. 배지 시스템**:
```
[ 헤어스타일 그리드 ]
┌──────┬──────┬──────┐
│ ⭐HOT │      │ 🆕NEW │
│ 포니테일│ 단발  │ 웨이브│
└──────┴──────┴──────┘
```

**2. 탭 분리**:
```
[전체] [인기] [최신]
```

**3. 정렬 옵션**:
```
정렬: [인기순 ▼] [이름순] [추가일순]
```

**주의**: "인기" 표시는 다양성을 해칠 수 있으므로 기본값이 아닌 옵션으로 제공

## 프리뷰 시스템

### 실시간 반영 속도

**사용자 기대치**:
- **즉각적 피드백** (< 100ms): 색상 변경, 슬라이더 조정
- **빠른 피드백** (< 500ms): 헤어스타일/의상 교체
- **허용 가능** (< 2초): 복잡한 3D 렌더링

**성능 최적화 기법**:

**1. LOD (Level of Detail) 시스템**:
```
프리뷰 모드: 낮은 폴리곤 메시 + 512px 텍스처
최종 렌더: 고해상도 메시 + 2048px 텍스처
```

**2. Debouncing**:
```typescript
// 슬라이더 드래그 중에는 프리뷰 업데이트 제한
const handleSliderChange = debounce((value) => {
  updatePreview(value);
}, 150); // 150ms 간격으로 업데이트
```

**3. Progressive Loading**:
```
1. 즉시 표시: 실루엣/윤곽선
2. 500ms 후: 저해상도 텍스처 적용
3. 완료 시: 고해상도 최종 렌더
```

**4. Texture Merging**:
- 여러 UV 맵을 하나로 병합하여 메모리 사용량 감소
- 캐릭터 로딩 시간 개선

### 프리뷰 뷰포트

**멀티 앵글 지원**:
```
[ 3D Viewport ]
회전 버튼: [정면] [45도] [측면] [후면]
줌: [+] [-] [리셋]
포즈: [기본] [액션] [앉기]
```

**카메라 컨트롤**:
- 마우스 드래그: 회전
- 스크롤: 줌
- 더블클릭: 리셋

**컨텍스트별 포커스**:
- **얼굴 편집 단계**: 자동으로 얼굴 클로즈업
- **의상 단계**: 전신 뷰로 전환
- **헤어 단계**: 머리 중심으로 프레임 조정

## Quick Start vs Advanced 분기

### 분기 전략

**1. 초기 질문으로 경로 결정**:
```
"어떻게 시작하시겠습니까?"

┌─────────────────────┐  ┌─────────────────────┐
│  빠른 시작          │  │  세부 커스터마이징  │
│  (추천 템플릿)      │  │  (모든 옵션)        │
└─────────────────────┘  └─────────────────────┘
```

**2. 템플릿 기반 Quick Start**:
- **프리셋 제공**: "전사", "마법사", "도적" 등 직업별 템플릿
- **원클릭 생성**: 랜덤 캐릭터 생성 → 세부 조정 시작점
- **AI 추천**: "당신의 스타일에 맞는 캐릭터" (선호도 기반)

**3. Progressive Disclosure**:
```
1단계: 핵심 5가지만 선택 (종족/성별/얼굴형/헤어/색상)
    ↓
"완료" 버튼 → 2단계로 이동
    ↓
2단계: 더 많은 옵션 (눈썹 조정, 문신, 장식 등)
```

### 온보딩 패턴

**초보자를 위한 가이드**:

**1. Tooltips & Contextual Help**:
```
[헤어스타일 옵션]
❓ 도움말: 헤어스타일은 캐릭터의 개성을 표현합니다.
          다양한 옵션을 시도해보세요!
```

**2. Interactive Tutorial**:
```
"첫 방문이신가요?"
→ [가이드 투어 시작] (2분 소요)
→ [건너뛰기]
```

투어 흐름:
- 1단계 하이라이트 + 설명
- 샘플 선택 자동 적용 → 프리뷰 변화 시연
- "다음" 버튼으로 진행

**3. Default Recommendations**:
```
각 카테고리의 첫 번째 옵션을 "추천"으로 표시
사용자가 선택하지 않으면 자동으로 추천 옵션 적용
```

**4. 점진적 복잡도**:
```
기본 모드: 5개 옵션만 표시
"고급 옵션 보기" 클릭 → 15개 옵션 추가 표시
```

### 온보딩 안티패턴

**❌ 시작 전 과도한 질문**:
```
"최악의 타이밍: 사용자가 시스템을 사용하기 전에 선택을 강요"
예: "알림 허용하시겠습니까?", "테마를 선택하세요" (사용 경험 없이)
```

**✅ 올바른 접근**:
```
기본값으로 시작 → 사용 중 필요 시점에 설정 권장
```

## 모바일 대응

### 터치 최적화

**1. 터치 타겟 크기**:
```
최소 크기: 44x44px (iOS 권장)
이상적: 48x48px (Android Material Design)
간격: 최소 8px
```

**2. 제스처 지원**:
```
스와이프: 이전/다음 단계 이동
핀치: 프리뷰 줌
탭: 옵션 선택
롱프레스: 상세 정보/미리보기
```

**3. 하단 시트 (Bottom Sheet) 활용**:
```
[ 캐릭터 프리뷰 - 상단 70% ]
┌─────────────────────────┐
│                         │
│    Character Preview    │
│                         │
└─────────────────────────┘
[ 옵션 선택 - 하단 30% ]
┌─────────────────────────┐
│ [헤어] [눈] [의상] [색상]│
│ ───────────────────     │ ← 위로 드래그 시 전체 확장
└─────────────────────────┘
```

### 모바일 UI 패턴

**카테고리 네비게이션**:
```
수평 스크롤 탭:
← [Body] [Face] [Hair] [Outfit] [Color] →
```

**옵션 표시**:
```
그리드 레이아웃 (3열):
┌────┬────┬────┐
│ 옵션│ 옵션│ 옵션│
│  1 │  2 │  3 │
├────┼────┼────┤
│ 옵션│ 옵션│ 옵션│
│  4 │  5 │  6 │
└────┴────┴────┘
```

**모바일 안티패턴**:
- ❌ 작은 슬라이더 (터치 어려움) → ✅ 큰 터치 영역 + 숫자 입력 옵션
- ❌ 드롭다운 메뉴 (탭 2회 필요) → ✅ 직접 선택 가능한 버튼 그리드
- ❌ Hover 효과 의존 → ✅ 탭으로 즉시 적용 또는 확인 버튼

## 베스트 프랙티스 요약

### UX 원칙

| 원칙 | 설명 | 구현 예시 |
|------|------|----------|
| **간결성** | 복잡한 작업을 3-5단계로 분해 | 10개 카테고리 → 4개 단계로 그룹핑 |
| **진행 상황 표시** | 현재 위치와 남은 단계 명확히 표시 | `[1/4] 기본 설정` + 진행 바 |
| **되돌리기 허용** | 언제든지 이전 단계로 돌아가기 가능 | 상단 스텝 인디케이터 클릭 → 해당 단계로 이동 |
| **기본값 제공** | 모든 옵션에 합리적인 기본값 설정 | "추천" 배지 + 인기 템플릿 |
| **맥락 제공** | 각 선택의 의미와 영향 설명 | 툴팁, 프리뷰 변화 실시간 반영 |
| **성능 최적화** | 500ms 이내 프리뷰 업데이트 | LOD, 텍스처 병합, Debouncing |

### 접근성 (Accessibility)

**1. 키보드 네비게이션**:
- Tab: 다음 옵션으로 이동
- Enter/Space: 선택
- Arrow Keys: 슬라이더/옵션 조정

**2. 스크린 리더 지원**:
```html
<button aria-label="헤어스타일: 포니테일 선택">
  <img src="ponytail.png" alt="" />
</button>
```

**3. 색상 의존도 낮추기**:
- 색상 외에 아이콘/텍스트로도 상태 표시
- 예: "선택됨" 체크마크 + 테두리 강조

## 안티패턴 및 주의사항

### 위저드 UI 안티패턴

| 안티패턴 | 문제 | 해결책 |
|---------|------|--------|
| **과도한 단계 수** | 10단계 이상 시 지루함/포기율 증가 | 3-5단계로 축소, 유사 항목 병합 |
| **정보 격리** | 이전 단계 정보 확인 불가 | 사이드바에 요약 표시, 스텝 클릭으로 이동 |
| **시기상조한 질문** | 시스템 사용 전 설정 강요 | 기본값 제공, 사용 중 변경 허용 |
| **진행 상황 숨김** | 몇 단계 남았는지 모름 | 상단에 `1/4` + 진행 바 표시 |
| **불명확한 질문** | 사용자가 무엇을 선택해야 할지 모름 | 명확한 라벨 + 예시 이미지 + 툴팁 |
| **2단계 위저드** | 위저드가 필요 없는 간단한 작업 | 단일 폼으로 대체 |
| **정보 비교 불가** | 단계 간 옵션 비교 어려움 | 요약 사이드바, 스텝 점프 허용 |

### 캐릭터 커스터마이저 특화 안티패턴

**1. 실시간 프리뷰 누락**:
```
❌ 옵션 선택 → "적용" 버튼 클릭 → 프리뷰 업데이트
✅ 옵션 선택 즉시 프리뷰 반영
```

**2. 고정된 카메라 앵글**:
```
❌ 정면 뷰만 제공 (뒤쪽 헤어스타일 확인 불가)
✅ 회전/줌 가능한 3D 뷰포트
```

**3. 색상 선택 UX 부재**:
```
❌ 드롭다운 메뉴에 "Red", "Blue" 텍스트만 표시
✅ 컬러 팔레트 그리드 + 커스텀 색상 선택기
```

**4. 초기화 기능 누락**:
```
❌ 잘못 선택 시 처음부터 다시 시작
✅ "기본값으로 초기화" 버튼
```

**5. 저장/불러오기 부재**:
```
❌ 한 번 생성하면 다시 편집 불가
✅ "프로필 저장" + "불러오기" 기능
```

## 사례 연구

### 1. ReadyPlayerMe (아바타 빌더)

**특징**:
- **Quick Start**: 셀카 업로드 → AI가 자동으로 3D 아바타 생성
- **Customization**: 얼굴형, 헤어, 의상, 액세서리 단계별 편집
- **플랫폼 통합**: Unity/Unreal Engine SDK 제공

**UX 강점**:
- 초보자: 30초 안에 아바타 완성 (셀카 → 완료)
- 고급 사용자: 세부 조정 옵션 제공
- 실시간 3D 프리뷰 (WebGL)

**구현 흐름**:
```
1. [셀카 업로드] → AI 얼굴 분석
2. [자동 생성된 아바타 프리뷰]
3. [커스터마이징 시작] (선택 사항)
   → 얼굴 → 헤어 → 의상 → 액세서리
4. [저장 및 내보내기]
```

### 2. Leonardo.ai (AI 캐릭터 생성)

**특징**:
- **Character Reference**: 참조 이미지 업로드 → 일관된 캐릭터 생성
- **강도 조절**: Low/Mid/High 슬라이더로 참조 영향도 조정
- **프롬프트 기반**: 텍스트 설명 + 이미지 가이던스 조합

**UX 강점**:
- 프롬프트 자동완성 및 예시 제공
- Elements (스타일 프리셋) 탭으로 빠른 적용
- 실시간 프리뷰 없음 (생성 버튼 → 4-10초 대기)

**워크플로우**:
```
1. [참조 이미지 업로드]
2. [프롬프트 작성] (예: "warrior with blue armor")
3. [스타일 프리셋 선택] (Anime, Realistic, Cinematic 등)
4. [Reference Strength 조정] (Low 30% / Mid 60% / High 90%)
5. [Generate] → 4장 생성 → 선택
```

### 3. Genshin Impact (게임 캐릭터 커스터마이징)

**특징**:
- **제한적 커스터마이징**: 플레이어 캐릭터는 고정, NPC/콜라보 캐릭터만 일부 수정 가능
- **팀 편성 중심**: 캐릭터 자체보다 파티 구성에 집중

**UX 관찰**:
- 캐릭터 디자인이 게임플레이와 직결 (원소, 무기 타입)
- "캐릭터 뽑기" 가챠 시스템 → 커스터마이징 불필요
- **교훈**: 커스터마이징이 항상 필요한 것은 아님 (게임 장르에 따라 다름)

### 4. FFXIV (심층 커스터마이징)

**특징**:
- **종족별 차별화**: 종족 선택 → 부족 선택 → 세부 옵션
- **슬라이더 기반**: 얼굴/몸 비율 미세 조정
- **광범위한 색상 선택**: RGB 팔레트 + 사전 정의된 색상

**UX 강점**:
- 프리셋 제공: "무작위 생성" 버튼으로 빠른 시작
- 비교 기능: 이전/다음 옵션 빠르게 전환
- 저장 슬롯: 여러 캐릭터 외형 저장 후 불러오기

**구조**:
```
┌──────────────────────┐
│ [종족] [부족] [성별] │ ← 기본 선택
├──────────────────────┤
│ [얼굴형] [헤어] [눈] │ ← 외형 카테고리
│ [피부색] [눈색]      │
├──────────────────────┤
│ [신체 슬라이더]      │ ← 고급 조정
│ • 키: ━━●━━━━       │
│ • 근육: ━━━━●━━     │
└──────────────────────┘
```

## Shorts Producer 적용 제안

### 현재 컨텍스트
- **대상**: AI 기반 쇼츠 영상 제작 플랫폼
- **캐릭터 용도**: 스토리보드에 등장하는 일관된 캐릭터 생성
- **기술 스택**: Stable Diffusion (이미지 생성) + Gemini (스토리보드)

### 권장 UX 패턴

**1. Quick Start 우선**:
```
┌────────────────────────────────┐
│ 새 캐릭터 만들기              │
│                                │
│ ┌──────────┐  ┌──────────┐   │
│ │ AI 자동  │  │ 직접     │   │
│ │ 생성     │  │ 커스터마이징│ │
│ │          │  │          │   │
│ │ [시작]   │  │ [시작]   │   │
│ └──────────┘  └──────────┘   │
└────────────────────────────────┘
```

**AI 자동 생성 흐름**:
1. 텍스트 프롬프트 입력 (예: "갈색 머리의 여성 전사")
2. Gemini가 태그 자동 생성
3. SD로 4개 후보 생성 → 선택
4. 선택한 캐릭터 저장 → 세부 조정 가능

**직접 커스터마이징 흐름** (3단계 위저드):
```
Step 1: 기본 설정
  • 나이대 (청소년/청년/중년/노년)
  • 성별 (남성/여성/중성)
  • 체형 (마른/보통/근육질/풍만)

Step 2: 외형
  • 헤어스타일 (10가지 프리셋 그리드)
  • 헤어 색상 (팔레트)
  • 눈 색상 (팔레트)
  • 피부 톤 (밝음/중간/어두움)

Step 3: 스타일
  • 의상 테마 (현대/판타지/SF/전통)
  • 표정 (중립/웃음/진지)
  • 포즈 (서있기/앉기/액션)

[생성 미리보기] → 4개 후보 → 선택 → 저장
```

**2. 프리뷰 전략**:
- **즉시 프리뷰 불가능**: SD 생성 4-10초 소요
- **해결책**: 단계별 선택 완료 후 "미리보기 생성" 버튼
- **로딩 UX**: 프로그레스 바 + 예상 시간 표시

**3. 카테고리 우선순위**:
```
P0 (필수): 나이/성별/헤어스타일/헤어색/피부톤
P1 (중요): 눈색/체형/의상 테마
P2 (선택): 표정/포즈/액세서리
```

**4. 모바일 대응**:
- 하단 시트 UI: 상단 프리뷰 + 하단 옵션 선택
- 그리드 레이아웃: 3열 (모바일) / 6열 (태블릿)
