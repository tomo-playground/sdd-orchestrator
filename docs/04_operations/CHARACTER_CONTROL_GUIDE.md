# 캐릭터 제어 및 일관성 가이드 (ControlNet & IP-Adapter)

Shorts Producer는 캐릭터의 외형 일관성을 유지하면서 다양한 장면을 연출하기 위해 **ControlNet**과 **IP-Adapter** 기술을 결합하여 사용합니다. 포즈 정보가 부족하거나 특정 장면 연출이 어려울 때 아래 가이드를 참조하세요.

## 1. 캐릭터 일관성 유지 (IP-Adapter)

IP-Adapter는 한 장의 **레퍼런스 이미지(Reference Image)**를 기반으로 캐릭터의 얼굴, 헤어스타일, 그림체 등을 모든 생성 결과물에 주입합니다.

- **작동 원리**: 프롬프트 Engine에 캐릭터의 시각적 특징을 직접 전달합니다.
- **설정**: 캐릭터 생성 시 등록한 `preview_image`가 레퍼런스로 사용됩니다.
- **팁**: 레퍼런스 이미지는 캐릭터의 전신이 보이고, 배경이 단순하며, 정면을 응시하는 이미지가 가장 효과적입니다.

## 2. 포즈 제어 (ControlNet OpenPose)

시스템은 프롬프트에서 `sitting`, `waving` 등 특정 키워드를 감지하면 미리 정의된 **골격 이미지(Skeleton)**를 사용하여 포즈를 강제합니다.

- **포즈가 부족할 때**: 
  1. `docs/04_operations/POSE_MAINTENANCE.md`를 참조하여 신규 포즈 골격 이미지를 추가합니다.
  2. 프롬프트에 직접적인 액션 태그(`fighting_pose`, `kneeling`, `looking_back`)를 사용하고 IP-Adapter 가중치를 높여 포즈를 유도합니다.

## 3. 포즈 데이터가 없을 때의 대응 (Reference-only)

특정 골격 정보가 없더라도 **Reference-only** 모드를 활용하면 레퍼런스 이미지의 구도와 스타일을 참고하여 자연스러운 결과물을 얻을 수 있습니다.

- **Reference-only 가중치 조절**:
  - 가중치가 높을수록(`0.7` 이상) 레퍼런스 이미지의 포즈를 그대로 따라갑니다.
  - 가중치가 낮을수록(`0.3` 이하) 프롬프트에 정의된 새로운 포즈를 더 자유롭게 생성합니다.

## 4. 장면 연출이 안 될 때의 체크리스트

1.  **IP-Adapter 모델 확인**: 애니메이션 캐릭터의 경우 `ip-adapter-plus_sd15` 모델이 설치되어 있어야 합니다.
2.  **프롬프트 충돌**: 프롬프트의 포즈 태그(예: `standing`)와 ControlNet 골격(예: `sitting`)이 충돌하면 이미지가 기괴하게 생성될 수 있습니다. 시스템의 자동 감지(`detect_pose_from_prompt`)를 믿거나 프롬프트를 정돈하세요.
3.  **캐릭터 레퍼런스 재생성**: 캐릭터의 특징이 잘 잡히지 않는다면, `Generation` 메뉴의 캐릭터 빌더를 사용하여 더 명확한 레퍼런스 이미지를 다시 생성(`Generate Reference`)하세요.
4.  **ControlNet 가중치 조절**: 장면 연출이 너무 경직되어 있다면 ControlNet Weight를 `0.8` 이하로 낮추어 프롬프트의 영향력을 높여보세요.

## 5. 고급: 신규 캐릭터 전신 포즈셋 구축

캐릭터의 일관성을 극대화하려면 해당 캐릭터에 특화된 포즈셋을 미리 생성해 두는 것이 좋습니다.
1. 캐릭터의 기본 레퍼런스 이미지를 생성합니다.
2. 해당 레퍼런스를 IP-Adapter로 고정한 상태에서 `standing`, `sitting`, `running` 등 주요 포즈별로 이미지를 배치 생성합니다.
3. 생성된 이미지들 중 가장 품질이 좋은 것들을 해당 캐릭터의 전용 에셋으로 관리합니다.
