# OutputTab UI Redesign Proposal

## 현재 구조 (Before)
```
┌─────────────────────────────────────┐
│ 채널 프로필                          │
│ ├─ 아바타 (❌ 로딩 실패)             │
│ ├─ 채널명: 레즈바이트                │
│ ├─ 캡션 입력                         │
│ └─ 좋아요 수 입력                    │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ Render Settings                     │
│ └─ Layout / BGM / Subtitle 등        │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ ▼ SNS OVERLAY                       │
│ ├─ Channel Profile 카드 (중복)      │
│ │  ├─ 아바타 (✅ 정상)              │
│ │  ├─ 채널명: 레즈바이트             │
│ │  ├─ Avatar: avatar_key            │
│ │  └─ Caption: 캡션 텍스트          │
│ ├─ Auto Fill (Legacy)               │
│ └─ Regenerate Avatar                │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ ▼ POST CARD META                    │
│ └─ (위와 동일한 구조 반복)           │
└─────────────────────────────────────┘
```

**문제점:**
- 정보 중복 (채널 프로필이 3곳에 표시)
- 아바타 로딩 방식 불일치 (상단: ❌, SNS/POST: ✅)
- 채널(고정) vs 영상(가변) 정보 혼재
- Auto Fill이 채널 프로필 의미를 퇴색

---

## 제안 A: 완전 분리 (권장)

### 구조
```
┌─────────────────────────────────────┐
│ 📺 채널 정보 (읽기 전용 카드)        │
│ ├─ 아바타 (50x50, 큰 사이즈)        │
│ ├─ 채널명: 레즈바이트                │
│ ├─ 프레임 스타일: Doremi             │
│ └─ [변경] 버튼 → 모달               │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ 🎬 이 영상 메타데이터                │
│ ├─ 캡션: [입력 필드] (60자)         │
│ │  └─ 50/60 (경고 표시)             │
│ └─ 좋아요: [입력 필드] (선택)        │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ Render Settings                     │
│ └─ Layout / BGM / Subtitle 등        │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ ▼ 렌더링 미리보기                    │
│ ├─ SNS Overlay 미리보기 이미지       │
│ │  (실제 오버레이 모습 시각화)        │
│ └─ Post Card 미리보기 이미지         │
│    (실제 포스트 카드 모습 시각화)     │
└─────────────────────────────────────┘
```

**장점:**
1. **명확한 정보 계층**
   - 채널 정보 = 고정 (한 번 설정 → 모든 영상 재사용)
   - 영상 메타 = 가변 (스토리마다 다름)

2. **중복 제거**
   - 채널 프로필을 1곳에서만 표시
   - SNS/POST 섹션은 "미리보기"로 전환

3. **UX 개선**
   - 채널 정보: 크고 읽기 쉬운 카드 (변경 드물음)
   - 영상 메타: 바로 접근 가능한 입력 필드 (매번 수정)
   - 미리보기: 최종 결과물 확인용

4. **아바타 일관성**
   - 1개의 channelAvatarUrl state만 사용
   - 모든 섹션에서 동일한 URL 참조

## 제안 B: 현재 구조 유지 + 버그 수정

### 변경 사항
1. 상단 "채널 프로필"에 channelAvatarUrl state 추가
2. useEffect로 channelProfile.avatar_key 변경 시 resolveAvatarPreview() 호출
3. SNS OVERLAY/POST CARD에서 "Channel Profile" 카드 제거 (Auto Fill만 유지)

**장점:**
- 최소 변경
- 기존 동작 유지

**단점:**
- 여전히 정보 중복
- 채널/영상 레벨 구분 불명확

---

## 추가 고려사항

### 아바타 재생성 위치
- **제안 A**: 상단 채널 카드에 [아바타 재생성] 버튼 추가
- **현재**: SNS OVERLAY 섹션 안에 숨어있음 (찾기 어려움)

### Auto Fill (Legacy) 버튼
- **문제**: 채널 프로필이 있는데 왜 Auto Fill?
- **제안**:
  - 채널 프로필 설정 시 Auto Fill 비활성화
  - 또는 "채널 프로필을 먼저 설정하세요" 안내만 표시

### 프레임 스타일 선택
- **현재**: channelProfile.default_frame_style (고정)
- **향후**: 스토리별로 프레임 변경 가능?
  - YES → 영상 메타데이터에 "프레임 스타일" 선택 추가
  - NO → 채널 프로필에만 유지

## 결정 필요

1. **제안 A vs 제안 B?**
   - A: 완전 분리 (권장, 작업량 중)
